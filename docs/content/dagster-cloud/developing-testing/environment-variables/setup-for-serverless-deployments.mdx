Next, uncomment the `env_vars` property in the repository's `/.github/workflows/deploy.yml` and `/.github/workflows/branch_deployments.yml` files:

## dagster-cloud CLI

If you're using a git provider like GitLab or Bitbucket, you can use the [dagster-cloud CLI](/dagster-cloud/developing-testing/dagster-cloud-cli) to pass in environment variables with the `--env` flag.

This flag accepts `KEY=VALUE` pairs. For example:

```shell
dagster-cloud serverless deploy --env GITHUB_API_TOKEN=TOKEN_VALUE
```
