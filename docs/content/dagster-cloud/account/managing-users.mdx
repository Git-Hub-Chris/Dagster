### Assigning user roles

With the exception of the **Organization Admin** role, user roles are set on a per-deployment, per-code location basis and enforced both in Dagster Cloud and the GraphQL API. Organization Admins have access to the entire organization, including all [full deployments](/dagster-cloud/developing-testing/managing-deployments), [code locations](/dagster-cloud/developing-testing/code-locations), and [Branch Deployments](/dagster-cloud/developing-testing/branch-deployments).

#### Deployment permissions

All Dagster Cloud accounts have the ability to set permissions at the deployment level. When defined, the selected user role will be used as the default for all code locations contained in the deployment.

For example, if a user is assigned the **Editor** user role for a deployment, they'll also be an **Editor** for all code locations in the deployment:

<Image
alt="Default Editor user role for prod code locations"
src="/images/dagster-cloud/user-token-management/editor-permissions.png"

/>

**Note**: Having access to a deployment doesn't automatically grant access to Branch Deployments. [Branch deployment permissions](#branch-deployment-permissions) must be granted separately.

#### Code location permissions

<Note>Code location permissions are a Dagster Cloud Enterprise feature.</Note>

Dagster Cloud Enterprise users can override the deployment-level user role by setting permissions at the code location level. **Note**: Code location permissions are disabled for **Organization Admins** and **Admins** at the deployment level, as these user roles are the most permissive.

- Disabled for: Org admins, and admins and editors at the deployment level.

- Also disabled for No Access at the deployment level

  - If a user has a less permissive role at the deployment level, only a more permissive grant is allowed at the code location level
  - If a user is granted a more permissive role at the deployment level, all code locations will use the new grant by default

- Removing overrides

## Adding users

<Note>
  <strong>Organization Admin</strong> or <strong>Admin</strong> permissions are
  required to add users in Dagster Cloud.
</Note>

Before you start, note that:

- **If using Google for SSO**, users must be added in Dagster Cloud before they can log in.

- **If using a SAML-based solution like Okta**, users must be assigned to the Dagster app in the SSO portal to log in. By default, users will be granted Viewer permissions on each deployment. The default role can be adjusted by modifying the [`sso_default_role` deployment setting](/dagster-cloud/developing-testing/deployment-settings-reference#sso-default-role).

1. Sign in to your Dagster Cloud account.
2. Click the **user menu (your icon) > Cloud Settings**.
3. Click the **Users** tab.
4. In the **Enter an email...** field, enter the user's email address.
5. Click **Add user**. The user will be added to the list of users.
6. In the window that displays, select the appropriate [user role](#understanding-user-roles) for each deployment:
   1. Next to a deployment, click **Edit user role**.
   2. Select the user role for the deployment. This [user role](#understanding-user-roles) will be used as the default for all code locations in the deployment.
   3. Click **Save**.
   4. **Enterprise only**: To set permissions for individual code locations in a deployment:
      1. Click the toggle next to the deployment.
      2. Click **Edit user role**.
      3. Select the user role for the code location.
      4. Click **Save**.
7. Repeat step 6 for each deployment.
8. **Optional**: To change the user's permissions for branch deployments:
   1. Next to **All branch deployments**, click **Edit user role**.
   2. Select the user role to use for all branch deployments.
   3. Click **Save**.
9. Click **Done**.

## Removing users

<Note>
  <strong>Organization Admin</strong> or <strong>Admin</strong> permissions are
  required to remove users in Dagster Cloud.
</Note>

Removing a user removes them from the organization. **Note**: If using a SAML-based SSO solution like Okta, you'll also need to remove the user from the Identity Provider (IdP). Removing the user in Dagster Cloud doesn't remove them from the IdP.

1. Sign in to your Dagster Cloud account.
2. Click the **user menu (your icon) > Cloud Settings**.
3. Click the **Users** tab.
4. Locate the user in the user list.
5. Click **Edit**.
6. Click **Remove user**.
7. When prompted, confirm the removal.
